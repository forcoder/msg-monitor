## 🎖️ 最佳实践规范

### 1. 测试编写原则
```markdown
✅ 每个方法都有对应的单元测试
✅ 边界条件全覆盖测试
✅ Mock外部依赖，测试内部逻辑
✅ 使用有意义的测试描述
✅ 保持测试独立性
```

### 2. 测试命名规范
```kotlin
@Test
fun `method description should be clear and specific`() {
    // Given - 清晰的数据准备
    // When - 明确的操作执行  
    // Then - 具体的验证结果
}
```

### 3. 测试数据管理
```kotlin
// 使用工厂方法创建测试数据
TestDataFactory.createAIModelConfig(isDefault = true)
TestDataFactory.createReplyHistory(styleApplied = false)
```

## 🔍 故障排除指南

### 常见问题解决
```markdown
#### 测试数据库连接失败
**症状**: Room database not found
**解决方案**: 检查@Database注解和实体类定义

#### MockWebServer无响应
**症状**: No responses from server
**解决方案**: 确保mock响应在测试前入队

#### 性能测试不稳定
**症状**: Flaky timing-based tests
**解决方案**: 使用固定时间模拟或增加容差阈值
```
