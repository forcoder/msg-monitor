## GitHub Actions CI/CD 工作流编写

为 csBaby Android 项目（客服小秘）编写了完整的 GitHub Actions 工作流，文件位于 `.github/workflows/`：

1. **`ci.yml`**：Push/PR 触发，运行 `lint` + 单元测试，传 `-PskipVersionIncrement -PskipUpload` 防止 CI 修改版本号
2. **`build-debug.yml`**：Push 到 main/develop 时自动构建 Debug APK，用 shell `sed` 递增版本号（Linux runner），构建完成后提交版本号变更（带 `[skip ci]`）
3. **`release.yml`**：打 `v*.*.*` tag 时触发，编译 Release APK → 上传 shz.al → 创建 GitHub Release；shz.al 密码从 `SHZAL_PASSWORD` Secret 读取，不再硬编码

附配置说明文档：`.github/CICD_GUIDE.md`

**关键设计决策**：
- Gradle 的 `incrementVersion` 任务仅供本地 `build.bat` 使用，CI 全程用 `sed` 处理版本递增，避免 Gradle 在 Linux 环境与 Windows 行为差异
- `skipVersionIncrement` 和 `skipUpload` 两个 Gradle 属性用于区分 CI 与本地构建

## 知识库功能优化

### 1. 弹窗知识库搜索增强

扩展了 `ReplyOrchestrator.searchKnowledgeRules()` 方法，增加了对回复模板、分类和目标名称的搜索功能。现在弹窗搜索与知识库页面的搜索范围完全一致：

- **关键词搜索**：通过 hybridSearch（关键词 + 语义）
- **回复模板搜索**：过滤 `replyTemplate.contains(query)` 的规则
- **分类搜索**：过滤 `category.contains(query)` 的规则
- **目标名称搜索**：过滤 `targetNames.any { it.contains(query) }` 的规则

合并结果去重后按评分排序，最多显示 15 条。

### 2. 知识库导入新增「追加」/「覆盖」模式

- **追加模式（默认）**：保留现有规则，直接插入新规则
- **覆盖模式**：先调用 `knowledgeBaseManager.clearAllRules()` 清空全部规则，再导入新规则

实现方式：
- `KnowledgeViewModel.kt`：新增 `ImportMode` 枚举 + `importRules(uri, mode)` 重载方法
- `KnowledgeScreen.kt`：新增 `ImportModeDialog` 对话框，用户先选模式再选文件，通过 `pendingImportMode` 状态变量传递模式选择

- `app/src/main/java/com/csbaby/kefu/presentation/screens/knowledge/KnowledgeViewModel.kt`
- `app/src/main/java/com/csbaby/kefu/presentation/screens/knowledge/KnowledgeScreen.kt`
