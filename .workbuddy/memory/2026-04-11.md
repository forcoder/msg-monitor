### 3. shz.al API 集成和自动版本升级

**新增文件**
- `ShzlApiService.kt` - shz.al API 服务接口
  - 上传 APK 接口
  - 检查更新接口
  - 获取最新版本接口
  - 获取版本列表接口
  - 专用响应类 ShzlApiResponse

- `upload-to-shzl.ps1` - 新的上传脚本
  - 上传 APK 到 https://shz.al/api
  - 自动读取版本信息
  - 自动生成更新说明
  - 保存版本信息和上传历史
  - 支持 MD5 校验
  - 友好的错误提示

**修改文件**
- `OtaRepository.kt` - 集成 shz.al API
  - 优先使用 shz.al API 检查更新
  - 失败时自动回退到模拟服务
  - 支持 ShzlApiResponse 数据格式

- `OtaAndOssModule.kt` - 添加 ShzlApiService 依赖注入
  - 提供 ShzlApiService 实例
  - 配置 baseUrl 为 https://shz.al/

- `build-and-upload.bat` - 更新编译上传流程
  - 移除阿里云 OSS 上传
  - 改用 upload-to-shzl.ps1 脚本
  - 自动版本升级（已在 build.gradle.kts 中配置）

**API 设计规范**
- POST /api/v1/apps/upload - 上传 APK
- GET /api/v1/apps/check-update - 检查更新
- GET /api/v1/apps/latest - 获取最新版本
- GET /api/v1/apps/versions - 获取版本列表

**工作流程**
1. 运行 `build-and-upload.bat`
2. 自动升级版本码和版本名
3. 编译 APK
4. 上传到 shz.al API
5. 客户端可直接检查更新并下载安装

**编译结果**

- APK: `app\build\outputs\apk\debug\app-debug.apk`
