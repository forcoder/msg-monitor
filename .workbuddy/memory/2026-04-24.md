# 2026-04-24 工作日志

## OTA下载崩溃修复（最终版）
- **问题**: 下载更新过程中应用秒退崩溃
- **根因** (共4个bug):
  1. **致命**: `setDestinationUri(Uri.fromFile(downloadFile))` 在targetSdk 34下，DownloadManager不接受`file://` scheme的URI，直接抛SecurityException崩溃
  2. **高**: `downloadOssUpdate()`在ViewModel中直接覆盖`updateStatus`为"正在下载OSS更新..."，与UI层的`contains("下载中")`判断不匹配，且与OtaManager的Flow监听状态冲突
  3. **中**: 下载广播回调中依赖`COLUMN_LOCAL_FILENAME`（Android 10+已废弃，返回null）
  4. **低**: `startDownload`没有保存update到`_availableUpdate`，导致广播回调中无法获取文件路径
- **修复**:
  - **OtaManager.kt**:
    1. `setDestinationUri(Uri.fromFile)` → `setDestinationInExternalFilesDir(context, null, "Updates/filename.apk")`
    2. 移除废弃的`COLUMN_LOCAL_FILENAME`逻辑，直接用`getApkFile()`获取文件，增加大小检查
    3. `startDownload`中增加`_availableUpdate.value = update`
  - **ProfileViewModel.kt**:
    1. `downloadOssUpdate()`不再直接覆盖updateStatus，改为让OtaManager统一管理状态
