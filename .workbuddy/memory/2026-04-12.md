# 2026-04-12 工作记录

## 大模型测试功能实现

### 修复的 Bug
- **Entity 缺少 `model` 字段**：`AIModelConfigEntity` 没有 `model` 列（域模型有，Entity 没有），导致用户填写的模型名（如 gpt-4）保存后丢失，测试时用了默认值 `gpt-3.5-turbo`，这是测试报错的根本原因。
- 修复：给 Entity 添加 `model` 字段 + EntityMapper 添加映射 + DB 迁移 MIGRATION_3_4（ALTER TABLE ADD COLUMN model TEXT NOT NULL DEFAULT ''）
- 数据库版本从 3 升到 4

### 新增功能
- **ModelEditDialog 增加"测试"按钮**：在添加/编辑模型对话框中可以直接测试连接，无需先保存。
- ViewModel 新增 `testConnectionWithConfig(config)` 方法，通过 `AIService.testConnection()` 直接测试。
- 新增 `DialogTestState` 密封类（Idle/Testing/Success/Error），管理对话框内的测试状态。
- 测试按钮显示 Loading 动画，成功/失败分别显示绿色/红色图标和消息。

### 测试报错修复（第二轮）
- **testConnection 验证逻辑修正**：原来检查 `config.modelName`（用户自定义名称），改为检查 `config.model`（实际模型 ID 如 gpt-4），因为发 API 请求用的是 `model` 字段
- **错误信息友好化**：新增 `friendlyErrorMessage()` 方法，将 401/403/404/429/500/超时/DNS 错误等技术性信息转为中文友好提示
- **通义千问 API 格式修正**：`buildTongyiRequest` 改为直接调用 `buildOpenAIRequest`（通义新版 API 已兼容 OpenAI 格式）；默认 API 地址从旧版 `dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation` 改为 `dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`
- **空响应处理**：测试时如果 API 返回空内容，给出"连接成功但模型返回空响应，请检查模型名称是否正确"的明确提示
